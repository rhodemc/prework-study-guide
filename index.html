HTML

  • HTML elements are usually composed of an opening and closing tag.
  • The head element contains site metadata unseen by visitors.
  • The body element contains visible content.
  • The link element connects external resources to the document.
  • The meta element defines data such as character set, description, keywords, author and viewport.
  • Common element types: text, image, header, list.
  • Elements that divide and describe: meta-related, content division, semantic.
  • Nested means code within code.
  • Elements nested inside other elements are considered Child Elements.
  • Element that holds the Child Elements is the Parent Element.

CSS

  • Inline CSS styles an element directly in the HTML file
  • Inline CSS is less flexible because each inline stlye must be edited individually.
  • Internal Style Sheet adds entire sheet to HTML file using a style element.
  • When using an Internal Style Sheet, rules only apply to that page.
  • External Style Sheets allow developers to keep all CSS rules in a separate file.
  • Use a link element to link an external style sheet.
  • Margin indicates how much space we want around the outside of an element.
  • Padding indicates how much space we want around the content inside an element.
  • A selector defines an element or attribute to which the rules or declarations will apply.
  • CSS Box Model includes margins outside the element, element borders, padding and content.
  • You can combine two selectors into a single CSS rule, promoting DRY (Do Not Repeat Yourself coding).

Git

  • git status: checks which branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds all modifications in current working branch to staging area.
  • git commit -m: creates a commit that captures current staged changes and includes a message.
  • git pull origin main: pulls current/updated main branch data. DO THIS OFTEN.
  • git push origin *branch name*: pushes local changes to remote branch and signals a pull request on GitHub.
  • Key to writing effective commit messages is to write them as imperative commands in the present tense.
  • A feature branch might be working on a single feature already incorporated into working version.
  • A develop branch might be developing features that haven't been incorporated into working version yet.
  • Base branch: branch we're merging into. Comparison branch: branch that needs merging.

JavaScript

  • JavaScript was invented by Brendan Eich.
  • We use conditional statements and loops to control the order in which code is executed or if it's executed at all.
  • Arrays are used to store data types.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script. Can deviate from flow if conditions are met.
  • You can add more than one condition by using else if.
  • Falsy: values like 0, "", null or Boolean - false.
  • Truthy: all other values including Boolean - true.
  • Strict Equality Operator: === checks to see if two values are equal. Returns true if values are equal, false if not.
  • If statements introduce decision-making into our code. Code will execute only if the statement meets a condition or if it's true.
